Pengertian Manajemen File pada Sistem Operasi Komputer

Manajemen File atau sering disebut dengan file sistem pada sistem operasi merupakan sebuah metode yang digunakan oleh sistem operasi komputer untuk mengorganisasikan file-file yang terdapat pada disk. Dengan adanya manajemen file akan mengurasi tingkat hilangnya file pada sistem akibat kelalaian kita baik disengaja maupun tidak disengaja. Misalnya data terhapus secara tidak disengaja selain itu dengan adanya manajemen file atau file sistem memudahkan kita untuk melakukan pencarian data.

Selain itu dengan manajemen file memungkinkan pengguna untuk dapat melakukan manajemen file dengan baik dan benar sehingga dapat menhemat media penyimpanan data dengan cara melakukan penghapusan data secara berkala data-data yang tidak terpakai.

>>Fungsi Manajemen File pada Sistem Operasi Komputer

>>Fungsi Manajemen File pada Sistem Operasi KomputerBerikut adalah beberapa fungsi manajemen file pada sistem operasi:

1.Pembuatan File, Memodifikasi File bahkan penghapusan file.

2.Penggunaan file secara bersama

3.Dapat digunakan untuk memback up dan merecovery data untuk mencegah terjadinya hilangnya data

4.Informasi dan data dapat disimpan dengan baik dan aman

>>.Sasaran Manajemen File pada Sistem Operasi Komputer

1.Manajemen file bertujuan untuk memenuhi kebutuhan dari manajemen data dari pengguna

2.Untuk menjamin data yang terdapat pada file

3.Optimasi kinerja sistem

4.Menyediakan dukungan untuk sistem input dan output dari berbagai macam media penyimpanan

5.Bertujuan meminimalisis kehilangan data atau rusaknya data

>>Arsitektur Manajemen File pada Sistem Operasi Komputer

1.Sistem akses yaitu berkaitan dengan cara penyimpanan data pada file akses

2.Manajemen FIle yaitu berkaitan dengan penyediaan ekanisme operasi pada file

3.Manajemen ruang penyimpanan yaitu berkaitan dengan alokasi file di dalam media penyimpanan

9. Berikut ini metode instalasi berdasarkan letak file sistem operasi disimpan beserta alasannya​.


Penjelasan:

1. Instalasi di partisi sistem (system partition)

Alasan: Instalasi di partisi sistem akan menjamin bahwa sistem operasi terinstall dengan sempurna dan dapat diakses dengan lancar oleh komputer. Selain itu, partisi sistem biasanya telah dipersiapkan dengan baik untuk menampung sistem operasi, sehingga proses instalasi akan lebih mudah dan cepat.

2. Instalasi di partisi lain (non-system partition)

Alasan: Instalasi di partisi lain dapat memberikan fleksibilitas untuk mengatur penggunaan ruang hard drive. Misalnya, jika Anda memiliki beberapa partisi yang terpisah, Anda dapat menginstall sistem operasi di salah satu partisi tersebut dan menyisakan partisi lain untuk penyimpanan data. Selain itu, instalasi di partisi lain dapat mempermudah proses backup sistem operasi dan data.

3. Instalasi di partisi terpisah (separate partition)

Alasan: Instalasi di partisi terpisah dapat memberikan keuntungan dari segi keamanan dan stabilitas sistem. Jika sistem operasi terinstall di partisi terpisah, maka kerusakan pada partisi lain tidak akan mempengaruhi sistem operasi. Selain itu, instalasi di partisi terpisah juga dapat mempermudah proses upgrade sistem operasi, karena Anda dapat mempertahankan data pada partisi lain saat menginstall sistem operasi yang baru.

21. Jelaskan jenis-jenis instalasi sistem operasi yang diketahui?


Instalasi Baru
Instalasi ini bisa digunakan apabila jaringan yang akan dibangun merupakan jaringan yang baru ataupun adanya penambahan perangkat server yang baru karena tak mendukung sistem operasi jaringan yang ada saat ini.

Apabila Anda memilih pilihan ini, maka semua data yang terdapat di partisi terpilih akan dihapus. Apabila ada aplikasi yang sebelumnya telah terinstall pada sistem operasi yang lama, maka aplikasi tersebut perlu untuk diintal kembali.


Upgrade
Pilihan ini banyak digunakan di dalam sistem-sistem jaringan yang telah berjalan. Pilihan ini dilakukan karena adanya perbaikan fitur yang ada pada sistem operasi yang digunakan, termasuk juga karena adanya fitur baru yang memang diperlukan.

Dengan memilih pilihan ini, aplikasi yang telah terinstall sebelumnya mungkin akan bisa digunakan setelah proses upgrade. Opsi upgrade hanya akan mengganti file-file sistem operasi sebelumnya dengan yang baru.

Multi-boot
Jika memang disyaratkan untuk memiliki adanya lebih dari 1 sistem operasi di dalam 1 komputer, maka pilihan ini memang pilihan yang tepat untuk memungkinkan penggunaan lebih dari 1 sistem operasi.

Nantinya, setiap sistem operasi tersebut akan ditempatkan di partisinya masing-masing. Maka dari itu, perlu adanya persiapan partisi yang cukup sebelum memulai untuk melakukan instalasi multi-boot ini.

Virtualisasi
Virtualisasi merupakan teknik yang memungkinkan adanya instalasi sistem operasi yang dilakukan di atas sistem operasi yang ada (yang berjalan) saat ini. Tidak dalam partisi yang tertentu, namun dalam suatu file yang tertentu.

File ini merupakan perwakilan dari suatu sistem komputer virtual. Satu komputer bisa memiliki lebih dari 1 sistem operasi dengan teknik virtualisasi ini. Beberapa aplikasi yang memungkinkan untuk membuat sistem virtual ini seperti VirtualBox, VMWare dan Virtual PC.
