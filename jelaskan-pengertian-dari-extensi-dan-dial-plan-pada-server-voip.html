Jawaban:

Untuk menggambarkan topologi ekstensi dan dial plan server softswitch, kita harus memahami dua komponen utama dalam arsitektur softswitch: server ekstensi dan server dial plan.

Server ekstensi adalah komponen yang bertanggung jawab untuk mengkoordinasikan ekstensi telepon yang digunakan oleh pengguna. Ekstensi telepon adalah nomor yang digunakan oleh pengguna untuk menghubungi satu sama lain atau untuk menghubungi sistem telepon lainnya. Server ekstensi dapat diimplementasikan sebagai aplikasi berbasis software atau sebagai perangkat keras yang terpisah.

Server dial plan adalah komponen yang bertanggung jawab untuk mengkoordinasikan routing panggilan telepon. Dial plan adalah aturan yang digunakan untuk menentukan di mana panggilan telepon harus diarahkan berdasarkan nomor yang ditelpon. Server dial plan dapat diimplementasikan sebagai aplikasi berbasis software atau sebagai perangkat keras yang terpisah.

Secara umum, topologi ekstensi dan dial plan server softswitch terdiri dari beberapa komponen utama, yaitu:

   Server ekstensi dan dial plan

   Gateway PSTN/PBX

   Jaringan IP atau VoIP

Server ekstensi dan dial plan dapat terhubung ke gateway PSTN/PBX melalui jaringan IP atau VoIP. Gateway PSTN/PBX digunakan untuk mengubah sinyal telepon analog menjadi sinyal digital yang dapat diteruskan melalui jaringan IP. Jaringan IP atau VoIP digunakan untuk mengirimkan sinyal telepon digital dari server ekstensi dan dial plan ke gateway PSTN/PBX.

Cara kerja dari topologi ekstensi dan dial plan server softswitch adalah sebagai berikut:

   Pengguna menghubungi ekstensi telepon yang ditentukan melalui telepon IP atau telepon analog

   Sinyal telepon diterima oleh server ekstensi dan dial plan

   Server ekstensi dan dial plan menentukan tujuan panggilan berdasarkan aturan dial plan yang telah ditentukan

   Server ekstensi dan dial plan mengirimkan sinyal telepon ke gateway PSTN/PBX

   Gateway PSTN/PBX meneruskan sinyal telepon ke tujuan yang ditentukan

   Panggilan telepon diterima oleh pengguna tujuan

6. jelaskan konsep konfigurasi dial plan di server softwitch


Konsep dari konfigurasi dial plan di dalam server softswitch adalah mengedit ekstensi dengan cara mengetik nano pada asterisk dial plan. Ketik perintah “Dial” pada bagian akhir dari file extensions.conf. Setelah itu tunggu selama beberapa detik. Setelah berhasil melakukan dial lakukan hangup, maka konfigurasi telah selesai dilakukan. Jangan lupa sebelum melakukan konfigurasi nonaktifkan dulu semua perintah dalam file ekstensi.

Pembahasan:

Dial plan adalah suatu program yang mengontrol adanya panggilan yang masuk atau keluar. Bukan hanya panggilan dial plan juga mampu menghubungkan antara jaringan sirkuit dengan jaringan paket seperti jaringan telepon tetap (PSTN), internet yang berbasis IP, TV kabel, serta jaringan seluler yang telah ada.  Dial plan mampu merutekan panggilan dari satu server ke server lain tanpa adanya gangguan. Hal ini dikarenakan dial plan mampu mengatur keluar masuknya server di dalam sambungan. Banyak sekali server yang ada saat ini, salah satu contohnya adalah server softswitch. Dial plan yang ada di dalam server softwitch akan mengatur setiap ekstensi untuk menghubungi sesama ekstensi atau dial trunk begitu juga sebaliknya. Dalam membuat dial plan kita harus menetapkan nomor telepon individu atau ekstensi masing-masing pengguna, dengan cara mengedit file extensions.conf yang terletak pada folder /etc/asterisk/.

8. sebut dan jelaskan 4 konsep dial plan


1. Context
Berisi beberapa instruksi extensions pada dial plan yang mempunyai beragam kegunaan.
Context ini membedakan interaksi dari satu group dengan group yang lain.
Sebuah extensi yang berada pada suatu context akan terisolasi dengan context yang lain.
Context dilutis dengan huruf besar atau kecil [A-Z] atau angkat [0-9]

2. Extensions
Dibangun dalam tiga komposisi:
Nama atau nomor extensi
Priority (setiap ekstension bisalebih dari satu priority)
Aplikasi (atau command) yang menampilkan aksi (action) dari call.
exten => nomor extensi, priority, aplikasi (action())
exten => 4000, 1, Answer()
exten => _4xxx,1,DIAL(ZAP/1,10,m)
exten => _4NNN,DIAL(DAHDI/g0,$EXTENT:1,20)

3.Priorities

4. Applications
Unnumbered Priority
n => next priority , diletakkan pada ext setelah priority 1, artinya tambah 1 dari priority sebelumnya

s => start extension , jk tidak ada nmr extension yang masuk makan extension ini yang akan dijalankan

t=> timeout, jika dalam interval terntu tidak ada masukkan maka extension ini akan dijalankan

10. Jelaskan konsep konfigurasi dial plan di server sofwitch


Jawaban:

Di dunia telekomunikasi, konfigurasi dial plan adalah pengaturan atau aturan yang digunakan untuk mengarahkan panggilan telepon dari satu titik ke titik lain di dalam sistem VoIP (Voice over Internet Protocol) atau sistem telepon lainnya yang menggunakan softswitch. Softswitch adalah perangkat lunak yang berfungsi sebagai pengendali lalu lintas panggilan dalam jaringan telekomunikasi.

Konfigurasi dial plan sangat penting dalam sistem telekomunikasi karena memungkinkan operator atau administrator jaringan untuk mengatur bagaimana panggilan harus ditangani dan diarahkan di dalam jaringan. Dengan menggunakan konfigurasi dial plan, administrator dapat menentukan bagaimana panggilan harus diteruskan, direlokasi, atau ditolak berdasarkan kriteria tertentu.

Berikut adalah beberapa konsep utama dalam konfigurasi dial plan di server softswitch:

1. **Pola Pencocokan (Pattern Matching):** Dalam dial plan, pola pencocokan digunakan untuk mengidentifikasi jenis panggilan tertentu berdasarkan digit yang dimasukkan oleh pengguna. Pola ini dapat berupa nomor telepon lengkap, awalan nomor, atau karakter lainnya. Misalnya, pola pencocokan mungkin digunakan untuk mengenali panggilan lokal atau internasional berdasarkan digit awalan tertentu.

2. **Rute dan Aksi:** Setelah pola pencocokan berhasil mengidentifikasi jenis panggilan, dial plan menentukan rute dan aksi yang harus diambil untuk panggilan tersebut. Rute ini menunjukkan jalur atau perangkat yang harus digunakan untuk mengarahkan panggilan. Misalnya, panggilan lokal dapat diarahkan ke saluran lokal, sedangkan panggilan internasional dapat diarahkan melalui gateway yang sesuai.

3. **Prioritas:** Dial plan biasanya berisi beberapa aturan yang mungkin cocok dengan satu panggilan. Untuk menentukan aturan mana yang harus diterapkan, dial plan menggunakan prioritas. Prioritas yang lebih tinggi menentukan aturan yang harus diikuti lebih awal dalam proses pencocokan. Jika satu aturan cocok dengan panggilan, dial plan akan menerapkan aturan tersebut tanpa memeriksa aturan lain yang mungkin cocok.

4. **Dampak dan Pembatasan:** Ketika mengkonfigurasi dial plan, perlu mempertimbangkan dampak dan pembatasan dari setiap aturan yang ditetapkan. Terlalu banyak aturan atau pemrosesan yang rumit dapat menyebabkan overhead dan mempengaruhi kinerja sistem.

5. **Transformasi dan Manipulasi Angka:** Dial plan juga dapat digunakan untuk mentransformasi atau memanipulasi angka dalam nomor telepon sebelum mengarahkan panggilan. Misalnya, penggunaan kode akses atau penambahan digit ekstra dapat dilakukan untuk mempersiapkan panggilan untuk dialing plan di jaringan selanjutnya.

6. **Pengelompokan dan Pengorganisasian:** Ketika dial plan kompleks dengan banyak aturan, penting untuk mengelompokkan dan mengorganisir aturan-aturan tersebut agar lebih mudah dikelola dan dipelihara.

Konfigurasi dial plan memungkinkan operator jaringan untuk mengatur sistem telekomunikasi agar dapat berfungsi dengan efisien dan sesuai dengan kebutuhan bisnis. Dengan mengelola dial plan dengan hati-hati, operator dapat memastikan bahwa panggilan ditangani dengan benar dan diarahkan ke tujuan yang tepat, mengoptimalkan kualitas layanan dan efisiensi jaringan.

Jawaban:

Konfigurasi dial plan pada server softswitch merupakan proses mengatur aturan dan kebijakan untuk mengarahkan panggilan telepon (voice call) yang masuk ke jaringan VoIP (Voice over Internet Protocol) atau layanan telepon berbasis IP. Server softswitch berfungsi sebagai pusat pengendali panggilan yang mengatur bagaimana panggilan tersebut harus diteruskan ke tujuan yang tepat.

Dial plan digunakan untuk mengenali pola nomor telepon tertentu, seperti nomor lokal, nomor nasional, nomor internasional, atau kode area tertentu. Kemudian, berdasarkan pola-pola tersebut, panggilan dapat diteruskan ke destinasi yang diinginkan. Konfigurasi dial plan di server softswitch melibatkan beberapa elemen utama, di antaranya:

1. **Patterns (Pola)**: Setiap dial plan berisi pola-pola numerik atau alfanumerik yang menentukan bagaimana nomor telepon diakui dan diproses. Pola tersebut mungkin mencakup kode area, panjang nomor, atau digit tertentu yang harus cocok agar dial plan diaktifkan.

2. **Routes (Rute)**: Setelah nomor dienali berdasarkan pola, dial plan perlu menentukan rute atau jalur yang harus diambil oleh panggilan tersebut. Ini bisa berupa jalur ke nomor internal dalam jaringan lokal, jalur keluar ke penyedia layanan telekomunikasi, atau pengalihan panggilan ke nomor lain dalam atau luar jaringan.

3. **Gateways (Gerbang)**: Dial plan perlu mengetahui cara menghubungkan panggilan ke jaringan PSTN (Public Switched Telephone Network) atau ke penyedia layanan VoIP eksternal. Untuk itu, server softswitch perlu dikonfigurasi dengan gerbang-gerbang atau gateways yang menghubungkan ke jaringan lain.

4. **Call Handling (Penanganan Panggilan)**: Konfigurasi dial plan juga mencakup bagaimana panggilan harus ditangani dalam situasi tertentu, seperti ketika nomor tidak valid, ketika sambungan sibuk, atau ketika terjadi panggilan konferensi.

5. **Time-based Rules (Aturan Berbasis Waktu)**: Server softswitch juga dapat mengizinkan konfigurasi aturan berbasis waktu, yang memungkinkan panggilan diarahkan ke tujuan yang berbeda berdasarkan jam tertentu atau hari tertentu.

Konfigurasi dial plan yang baik sangat penting untuk menyediakan layanan telepon yang andal, efisien, dan sesuai dengan kebutuhan pengguna. Dengan dial plan yang tepat, panggilan dapat diarahkan dengan akurat, meminimalkan biaya, dan meningkatkan kualitas layanan.

1. Konfigurasi Ekstensi Server Softwitch

Untuk melakukan konfigurasi ekstensi pada server softswitch, kita harus melakukakn pengeditan pada file sip.conf.

Ada dua hal yang harus dikonfigurasi pada file itu, yaitu konfigurasi user umum dan konfigurasi user data atau extension.

[general] -->> context umum,harus ada

;softphone (nama atau nomor account)

username=101 (login account)

nat=no (tanpa NAT)

context=tkj (context jaringan user)

canreinvite=no (mekanisme canreinvite)

mailbox=101@tkj (username@contex)

23. jelaskan bagian bagian dalam file konfigurasi extension.conf dalam server voip asterisk​


JAWABAN:

Agar lebih mudah untuk memahami apa itu ekstensi dan apa itu dial plan pada server Voip.

Ekstansi dan dial plan server Voip dianalogikan seperti PAB

Ekstansi adalah data client Voip, misalnya komputer ke 1 ekstansinya 001 , komputer ke 2 ekstansinya 002 , komputer ke 3 ekstansinya 003 dan seterusnya.

Semua ekstensi diatas ditulis dalam sebuah file konfigurasi yaitu file extensions conf. Setiap ekstensi memiliki komponen dan tahapan perintah yang tersimpan didalam file tersebut. Didalam file tersebut ditulis dalam format exten = Extensions, priority, Command [ parameter ]. Untuk setiap extensions / ekstensi, dapat didefinisikan sekumpulan perintah. Komponen yang membangun tahapan perintah extension atau Command line adalah sebagai berikut.

1.] Ekstension adalah label dari extension, Dapat berupa sebuah string [ angka, huruf dan simbol yg diijinkan ] Atau pola yg harus di evaluasi secara dinamik untuk dengan banyak kemungkinan nomor telepon, Setiap command line yang menjadi bagian dari extension tertentu harus mempunyai label yg sama.

2.] Priority, biasanya berupa angka integer merupakan urutan dari perintah yg harus dijalankan dalam sebuah extension. Perintah pertama yang akan di jalankan harus dimulai dengan prioritas 1, jiia tidak ada prioritas 1, maka asterisk tidak akan menjalankan perintah extension setelah prioritas 1 dijalankan, Asterisk akan menambah prioritas ke prioritas 2 dan seterusnya, Tentunya jika tidak ada perintah yg menentukan prioritas mana yg selanjutnya harus dijalankan. Jika ternyata perintah selanjutnya ternyata tidak terdefinisikan maka Asterisk akan menghentikan proses menjalankan perintah walaupun masih ada perintah dengan prioritas yg lebih tinggi.

33. apa hubungan antara dial plan dan ekstensi


Jawaban:

Dial plan dan ekstensi adalah dua konsep yang terkait dalam sistem telepon VoIP (Voice-over-IP). Dial plan adalah seperangkat aturan dan prosedur yang digunakan untuk menentukan cara panggilan diteruskan dari satu tempat ke tempat lain. Ekstensi pada dasarnya adalah nomor telepon yang digunakan untuk mengidentifikasi setiap stasiun telepon dalam jaringan VoIP.

Dalam sistem telepon VoIP, setiap stasiun telepon biasanya diberi nomor ekstensi yang unik. Nomor ekstensi ini digunakan untuk mengidentifikasi stasiun telepon di dalam jaringan dan memungkinkan panggilan untuk diteruskan langsung ke stasiun telepon yang dituju. Nomor ekstensi juga dapat digunakan untuk menentukan cara panggilan diteruskan dalam sistem dial plan.

Dial plan biasanya terdiri dari beberapa elemen, termasuk:

1. Prefixes (Awalan): Awalan digunakan untuk menunjukkan jenis panggilan yang akan dilakukan, seperti panggilan lokal, panggilan jarak jauh, panggilan internasional, dan sebagainya.

2. Patterns (Pola): Pola digunakan untuk menentukan nomor telepon mana yang akan diteruskan ke stasiun telepon tertentu. Pola dapat berupa nomor telepon atau kumpulan digit yang terkait dengan nomor telepon.

3. Routes (Rute): Rute digunakan untuk menentukan cara panggilan diteruskan. Hal ini dapat mencakup rute langsung ke stasiun telepon tertentu, rute melalui jaringan internal atau eksternal, atau rute melalui operator.

Dalam sistem VoIP, ekstensi digunakan untuk mengidentifikasi stasiun telepon yang akan menerima panggilan. Setiap nomor ekstensi harus disesuaikan dengan dial plan yang telah ditentukan untuk memastikan panggilan diteruskan dengan benar. Misalnya, jika nomor ekstensi 1000 digunakan untuk mengidentifikasi stasiun telepon tertentu, maka dial plan harus menyertakan pola yang sesuai dengan nomor tersebut untuk memastikan panggilan diteruskan ke stasiun telepon yang tepat.

Dengan demikian, hubungan antara dial plan dan ekstensi adalah bahwa dial plan harus mencakup pola yang mencakup nomor ekstensi yang digunakan di jaringan VoIP untuk memastikan panggilan diteruskan dengan benar ke stasiun telepon yang dituju.
